# 蜃灵助手插件项目结构

本文件用于新窗口快速理解蜃灵助手插件的代码构成。开新窗口时，建议先读本文件，再按任务阅读具体模块，避免每次全量扫代码。

当前插件版本见 `manifest.json`。如果修改插件代码，需要同步提升 `manifest.json` 小版本号；只改文档一般不需要提升版本号。

## 项目入口

```text
ShenLing-Extension/
├─ manifest.json        # SillyTavern 插件声明，版本号在这里
├─ index.js             # 插件总入口、主面板、模块路由、通讯日志、初始化
├─ style.css            # 全局 UI、主面板、日记本、移动端样式
├─ README.md
├─ 项目结构.md          # 本文件
├─ assets/
└─ src/
```

## 入口文件职责

### `manifest.json`

SillyTavern 第三方插件声明。

重要字段：

- `display_name`：插件显示名。
- `js`：入口脚本，目前为 `index.js`。
- `css`：样式文件，目前为 `style.css`。
- `version`：插件版本。修改插件代码后必须提升小版本号。
- `homePage`：GitHub 仓库地址。

### `index.js`

插件总入口，负责：

- 创建/刷新蜃灵助手浮窗。
- 管理左侧模块导航。
- 渲染设置页、API 设置、通讯日志。
- 注入各功能模块的配置依赖。
- 注册自动总结、情感档案、世界书缓存等事件监听。
- 调用各模块的 `render...Panel()` 与 `bind...Events()`。

常见修改场景：

- 新增一个主面板模块入口。
- 给现有模块接入全局依赖。
- 调整通讯日志 UI。
- 调整主面板整体交互。

注意：

- 不要把具体业务逻辑塞回 `index.js`。
- 新功能应放到 `src/features/模块名/` 或 `src/core/`。

### `style.css`

所有插件样式目前集中在这里，包括：

- 主面板布局。
- 移动端适配。
- 设置页与通用控件。
- 自动总结、情感档案、词汇替换、日记、小剧场模块样式。
- 日记本弹窗、封面、内页、手机全屏样式。
- 小剧场收藏夹、生成页、预览弹窗、深浅主题样式。

常见修改场景：

- UI 美化。
- 手机端布局问题。
- 日记本封面/内页表现。
- 按钮、卡片、表单样式。

注意：

- 日记本弹窗是独立沉浸式体验，不要把它塞回普通主面板布局。
- 手机端日记内页可能使用手机专用素材变量。

- 默认大总结模板。
- 蜃灵前置/支持 messages。

常见修改场景：

## 工具层

### `src/utils/text.js`

通用文本工具。

主要职责：

- 深拷贝与默认值合并。
- HTML 转义。
- 时间格式化。
- tag 列表解析。
- 总结正文净化与标签提取。

常见调用：

- 自动总结读取正文。
- 情感档案计算指纹。
- UI 输出转义。

### `src/core/api.js`

API URL 与模型列表工具。

主要职责：

- 规范 `/v1` 地址。
- 拼接 chat completions URL。
- 拼接 models URL。
- 解析模型列表。
- API 模式显示名。

### `src/core/chat.js`

SillyTavern 聊天读写兼容层。

主要职责：

- 获取 ST context。
- 安全读取聊天楼层。
- 读取指定楼层。
- 设置楼层内容。
- 创建 assistant 楼层。
- 隐藏/显示楼层。
- 生成楼层范围。

常见修改场景：

- ST 版本兼容问题。
- 聊天楼层读写异常。

注意：

- 这是底层兼容层，改动需谨慎。

### `src/core/settings.js`

全局设置与聊天 metadata 存储。

主要职责：

- 默认全局设置。
- 默认聊天状态。
- 获取/保存全局设置。
- 获取/保存当前聊天状态。
- 存储诊断。
- 获取 summary / emotion / word replace / chat beautify / mini theater 设置。

当前聊天专属数据包括：

- 自动总结状态。
- 归档记录。
- 情感档案。
- 日记本数据。
- 小剧场收藏结果。

全局数据包括：

- API Profile。
- 词汇替换规则。
- 小剧场提示词、文风、文件夹。
- UI 偏好。

注意：

- 聊天专属数据写入当前聊天 metadata。
- 全局设置写入 ST extension settings。

### `src/core/macros.js`

提示词宏替换。

主要职责：

- 替换 `{{user}}`。
- 替换 `{{char}}`。
- 替换 `{{original}}`。
- 替换 `<user>`、`<char>`、`<original>`。
- 解析 `{{random::A::B::C}}` 随机宏。
- 支持 messages 数组批量替换。

注意：

- 插件提示词发送前一般走本地预处理替换。
- 裸写的 `char` 不替换，避免误伤世界名或普通英文内容。

### `src/core/logs.js`

通讯日志工具。

主要职责：

- 创建通讯日志结构。
- 脱敏 API Key。
- 格式化 messages / 请求体 / 响应。
- 记录替换前解析、替换前正文、禁词替换结果。
- 复制日志文本。

当前使用：

- 日记、小剧场生成会记录 Context Resolver 本次读取素材。
- 日记、小剧场生成会记录禁词替换是否生效。

### `src/core/character.js`

当前角色卡基础信息读取。

主要职责：

- `getCurrentCharacterInfo()`
- `buildCharacterFoundationBlock()`

当前使用：

- 0 楼小总结会使用 `buildCharacterFoundationBlock()`。

注意：

- Context Resolver 里也有 `getResolvedCharacterCard()`，会做宏替换。
- 后续可把角色基础信息 formatter 统一到 Context 层，但不要破坏 0 楼总结现有行为。

### `src/core/summary.js`

总结核心工具与提示词组装。

主要职责：

- 提取/规范 `<memory>`。
- 提取/规范 `<grand_memory>`。
- 构造小总结 prompt。
- 构造大总结 prompt。
- 构造总归档压缩 prompt。
- 构造旧聊天归档批次 prompt。
- 解析 OpenAI 风格响应。

注意：

- 这里是总结提示词拼接核心。
- 不要把日记/小剧场的完整上下文拼接逻辑塞进这里。

## Context Resolver

### `src/core/context-resolver.js`

蜃灵公共上下文层，已完成第一轮正式化。

当前职责：

- 读取当前角色卡。
- 读取用户 Persona。
- 读取最近聊天。
- 读取最近 memory。
- 读取最近 grand_memory。
- 读取情感档案最新版。
- 捕获酒馆世界书激活缓存。
- dry run 世界书兜底。
- 过滤世界书系统/MVU/状态栏/输出格式类条目。
- 返回 diagnostics。
- 生成日记上下文材料。
- 生成小剧场等通用梦境上下文材料。

核心函数：

- `getResolvedCharacterCard()`
- `getUserPersona()`
- `collectRecentChatMessages()`
- `formatRecentChatForPrompt()`
- `collectRecentMemories()`
- `collectRecentGrandMemories()`
- `collectEmotionProfiles()`
- `collectCachedWorldInfoContext()`
- `collectDryRunWorldInfoContext()`
- `collectWorldInfoContext()`
- `registerWorldInfoContextEvents()`
- `formatShenlingContextForPrompt()`
- `resolveShenlingContext()`
- `buildDiaryContextMaterial()`
- `resolveDiaryContext()`

重要边界：

- 日记、小剧场、七日程、平行事件、剧情规划适合走完整 Resolver。
- 总结模块不应默认接完整 Resolver。
- 情感档案是 Resolver 的素材来源之一，不是完整 Resolver 消费者。
- dry run 是兜底和调试，不是正式用户功能。

当前状态：

- 公共基础读取函数已规范。
- formatter 已拆出供日记、小剧场复用。
- 世界书缓存优先、dry run 兜底、过滤 diagnostics 已稳定。
- diagnostics 已进入日记/小剧场通讯日志。
- 后续模块按需接入，不强行一次性接满完整 Resolver。

#### `workflow.js`

自动总结、大总结、旧聊天归档、总归档压缩的业务流程。

主要职责：

- 自动小总结监听。

- 重新生成大总结。
- 旧聊天归档。
- 总归档压缩。

特别警戒：

- 总结模块有自己的专用素材链路：
  - `createSummarySourceMaterial()`

  - `buildArchiveMemoryMaterial()`
- 不要让自动小总结默认读取完整世界书、Persona、情感档案、最近剧情包。
- 总结是事实压缩，背景素材太多容易污染事实。

#### `panel.js`

自动总结 UI。

主要职责：

- 总结开关。
- 提示词设置。
- 正文读取规则。
- 大总结/总归档设置。
- 归档管理器。
- 0 楼/指定楼层工具。
- memory/grand_memory 编辑器。

### `src/features/emotion-profile/`

```text
emotion-profile/
├─ workflow.js
└─ panel.js
```

#### `workflow.js`

情感档案业务逻辑。

当前运行方式：

1. 小总结 prompt 追加情感判断。
2. AI 返回 `<memory>` + `<emotion_update>`。
3. 插件解析 `<emotion_update>`。
4. 转写可读 `<emotion>` 块进 `<memory>`。
5. 先存入 `pendingByMessage`。
6. 下一轮发送前按当前 swipe 指纹确认。
7. 最新状态通过 `setExtensionPrompt()` 注入主 API 上下文。

重要边界：

- 情感档案生成不调用世界书缓存。
- 情感档案生成不调用 dry run。
- 情感档案不独立发送一轮 API，而是挂在小总结/大总结结果后解析。

#### `panel.js`

情感档案 UI。

主要职责：

- 显示当前待确认情感变化。
- 显示历史情感档案。
- 编辑待确认/正式档案。
- 切换情感档案开关。

### `src/features/diary/`

```text
diary/
└─ panel.js
```

日记本系统。

当前能力：

- 角色日记本。
- 普通角色日记。
- 交换日记。
- 日记草稿/收录。
- 编辑、删除、导出。
- 封面/内页预设。
- 自定义封面/内页上传。
- 桌面端双页日记本。
- 手机端全屏单页/长页体验。
- 调用 `resolveDiaryContext()` 生成上下文素材。

关键设计：

- 普通日记和交换日记统一放在同一本角色日记本。
- 用户内容为空时生成普通角色日记。
- 用户写了日记内容时生成交换日记回复。
- 日记不写入聊天楼层，只保存当前聊天 metadata。
- 日记默认跟随当前聊天，删除聊天则日记也消失。

注意：

- 日记模块目前主干完成，但仍需要多轮实测。
- 日记 UI 美化和素材适配可能继续调整。
- 不要把日记本弹窗改回普通主面板卡片。

### `src/features/plot-outline/`

```text
plot-outline/
├─ workflow.js
└─ panel.js
```

剧情大纲模块（第二阶段生成 API）。

当前能力：

- 模块入口挂载（复用 `outline` 导航格）。
- chatState 存储：`enabled / userDirection / storyCore / chapters / currentChapterId / progress / updatedAt`。
- 全局设置：`modules.plotOutline.apiMode / chapterCount`。
- 通过 `resolveShenlingContext()` 收集角色、世界书、近况、总结与情感档案素材。
- 调用主 API 或副 API 生成严格 JSON 章节蓝图。
- 生成结果先进面板草稿，用户保存后才写入当前聊天 metadata。
- 草稿和已保存大纲均支持表单式编辑。
- 当前章节可手动切换，推进条件可手动勾选修正。
- 清空会删除整个大纲；重置只清空 progress 并回到第一章。
- 通讯日志记录 messages、contextDiagnostics、响应、解析结果与禁词替换情况。

关键设计：

- 完整大纲数据保存在 chatState.outline，不塞进正文上下文。
- getter `getPlotOutlineState()` 负责结构规范化。
- 生成 workflow 放在 `workflow.js`，panel 只负责 UI、草稿和用户交互。
- `exitChapterId` 不信任模型输出，规范化时强制顺序链（CH01→CH02→…→末章为空）。

注意：

- 本阶段仍不做正文注入，不解析 `[progress:...]`，不自动切章。
- 章节字段为 `{ id, title, stage, theme, synopsis, keyEvents, conditions, exitChapterId }`。
- `conditions` 每条格式为 `{ id: 'C1', text: '' }`。

### `src/features/mini-theater/`

```text
mini-theater/
└─ panel.js
```

小剧场模式，替代原灵感工具。

当前能力：

- 小剧场提示词收藏。
- 文风收藏。
- 文件夹、搜索、排序。
- 收藏提示词复制、编辑、删除。
- 收藏提示词一键放入生成页。
- 收藏提示词纸飞机按钮发送到酒馆聊天输入框，由用户自行发送到聊天楼层。
- 主/副 API 切换。
- 独立生成小剧场番外。
- 调用 `resolveShenlingContext()` + `formatShenlingContextForPrompt()` 生成上下文素材。
- 支持文字、Markdown、HTML/CSS 结果预览。
- HTML 结果使用隔离 iframe 预览。
- 预览页支持编辑、保存、复制、收藏/取消收藏。
- 回看页支持已收藏小剧场打开、编辑、复制、删除。
- 生成结果支持禁词替换，并在 toast 与通讯日志中记录替换情况。
- 深浅主题同步到主界面、弹窗与预览外壳。

关键设计：

- 小剧场是主线外番外，不自动写入聊天楼层。
- 插件内独立生成的结果绑定当前角色与当前聊天，保存到当前聊天 metadata。
- 用户收藏的提示词和文风是全局素材，不默认绑定角色卡。
- 纸飞机按钮只把提示词放入酒馆输入框，不自动发送。

注意：

- 小剧场提示词在 `src/prompts.js`，不要散回 panel。
- HTML 预览不要强行改写模型生成页面的内部样式。
- 手机端收藏/生成/回看仍在细节打磨阶段。

## 后续计划模块

### 七日程

依赖 Context Resolver。

定位：

- 用户确认制。
- 生成草稿、重 Roll、编辑、确认保存。
- 默认不自动写入主线。

### 平行事件

定位：

- 低频番外事件。
- 可手动生成。
- 后续可能写入 `<DNote>`。

### 剧情规划 / 逆攻略

定位：

- 更接近主线生成辅助。
- 风险更高，建议等 Context、小剧场、七日程更稳定后再做。

## 新窗口推荐阅读顺序

如果是普通功能修改：

1. 读本文件。
2. 读相关模块文件。
3. 只按需读 `index.js` 的模块挂载位置。
4. 最后看 `style.css` 的相关 CSS 块。

如果是 Context Resolver 改造：

1. 本文件。
2. `src/core/context-resolver.js`。
3. `src/features/diary/panel.js`。
4. 必要时读 `src/features/summary/workflow.js`，但不要直接改总结 prompt。

如果是日记 UI：

1. 本文件。
2. `src/features/diary/panel.js`。
3. `style.css` 中 `slx-diary` 相关块。
4. `assets/diary/` 素材尺寸。

如果是自动总结：

1. 本文件。
2. `src/features/summary/workflow.js`。
3. `src/core/summary.js`。
4. `src/prompts.js`。
5. `src/features/summary/panel.js`。

如果是情感档案：

1. 本文件。
2. `src/features/emotion-profile/workflow.js`。
3. `src/features/emotion-profile/panel.js`。
4. `src/prompts.js` 的情感档案提示词段。

如果是小剧场：

1. 本文件。
2. `src/features/mini-theater/panel.js`。
3. `src/prompts.js` 的 `buildMiniTheaterPrompt()`。
4. `src/core/context-resolver.js` 的通用 formatter。
5. `style.css` 中 `slx-theater` 相关块。

如果是剧情大纲：

1. 本文件。
2. `同步脚本/预设/蜃灵/蜃灵助手剧情大纲七日程平行事件计划.md`（施工计划与设计备忘）。
3. `src/features/plot-outline/workflow.js`。
4. `src/features/plot-outline/panel.js`。
5. `src/prompts.js` 的 `buildPlotOutlinePrompt()`。
6. `src/core/settings.js` 的 `getPlotOutlineState()` 与 `defaultChatState.outline`。
7. `style.css` 中 `slx-outline-` 相关块。

## 施工注意事项

- 修改插件代码后，提升 `manifest.json` 小版本号。
- 不要把业务逻辑重新塞回 `index.js`。
- 不要让总结模块默认吃完整 Context Resolver。
- 不要让情感档案生成默认接世界书或 dry run。
- 世界书 dry run 是兜底/诊断，不是正式用户功能。
- 日记、小剧场、七日程等番外/规划类模块可以吃完整 Context Resolver。
- 用户个人备注在外部计划 MD 中，不要改动。
- 如果遇到旧“蜃灵切换”脚本任务，改源文件 `src/蜃灵切换` 后运行 `pnpm build`，不要直接改 dist。
